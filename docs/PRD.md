## 1. 개요

### 1.1 제품 비전

Voice Ledger는 개인용 macOS 데스크톱 앱으로, 음성 녹음 파일(m4a/mp4)을 완전 오프라인 환경에서 타임스탬프 포함 텍스트(.txt)로 변환한다. 네이버 클로바 노트의 핵심 기능(음성→텍스트 변환)을 외부 API 없이 로컬에서 수행한다.

### 1.2 핵심 가치

- **프라이버시:** 녹음 파일이 외부 서버로 전송되지 않음
- **오프라인:** 인터넷 연결 없이 언제든지 사용 가능 (최초 모델 다운로드 제외)
- **단순성:** 드래그앤드롭 하나로 변환 시작, 별도 설정 불필요

## 2. 사용자 정의

### 2.1 타겟 사용자

- 개인 사용자 (1인)
- macOS M4 (Apple Silicon) 사용
- 한국어 및 영어 녹음 파일 보유
- 회의 녹음, 인터뷰, 음성 메모 등을 텍스트로 정리 필요
- 기술적 배경 없어도 사용 가능한 수준의 UX 기대

### 2.2 사용 시나리오 (구체화)

#### 시나리오 A: 회의 녹음 변환 (주요 사용 사례)
1. 사용자가 아이폰 음성 메모 앱으로 1시간짜리 팀 회의를 녹음 (.m4a)
2. AirDrop으로 Mac에 파일 전송 (`~/Downloads/meeting_20260405.m4a`)
3. Voice Ledger 앱을 열어 파일을 드래그앤드롭
4. 진행 바를 확인하며 25분 이내에 변환 완료
5. `~/Downloads/meeting_20260405.txt` 파일에서 타임스탬프별 발언 내용 확인
6. 텍스트 파일을 Notion/메모장에 붙여넣어 회의록 정리

#### 시나리오 B: 영상 회의 녹화 파일 변환
1. Zoom 로컬 녹화 파일 (`zoom_recording.mp4`)이 있음
2. 파일을 드래그앤드롭 → 앱이 mp4에서 오디오 트랙 자동 추출
3. 영어/한국어 혼합 발언이 모두 올바르게 변환됨
4. `zoom_recording.txt` 파일 생성 확인

#### 시나리오 C: 최초 실행 (모델 다운로드)
1. 설치 후 첫 실행 시 Whisper medium 모델 (~1.5GB) 다운로드 필요
2. 앱이 다운로드 진행률과 예상 시간을 명확히 표시
3. 다운로드 완료 후 즉시 파일 변환 가능

#### 시나리오 D: 이미 변환한 파일 재변환
1. 사용자가 동일 파일을 다시 드래그앤드롭
2. 동일 위치에 `.txt` 파일이 이미 존재
3. 앱이 덮어쓰기 여부를 사용자에게 확인 후 처리

## 4. 비기능 요구사항

### 4.1 성능

| 기준 | 목표 | 측정 방법 |
|------|------|----------|
| 1시간 파일 변환 시간 | 25분 이내 | M4 Mac, medium 모델, MPS 가속 |
| 30분 파일 변환 시간 | 12분 이내 | 동일 환경 |
| 앱 시작 시간 | 5초 이내 | 콜드 스타트, 모델 로딩 제외 |
| 모델 로딩 시간 | 30초 이내 | 최초 변환 시작 시 |
| 메모리 사용량 | 4GB 이내 | 변환 중 최대값 기준 |
| 앱 유휴 메모리 | 500MB 이내 | 변환 완료 후 대기 상태 |

### 4.2 호환성

- macOS 13 (Ventura) 이상
- Apple Silicon (M1/M2/M3/M4) 우선 지원, MPS 가속 활성화
- Intel Mac: 동작하나 CPU 변환으로 속도 느림 (성능 보장 없음)
- 동시 실행 인스턴스: 1개 (단일 인스턴스 권장, 다중 실행 시 동작 미보장)

### 4.3 설치 및 배포

- **단기 (개발용):** Python 가상환경 + `requirements.txt` 방식
- **장기 (배포용):** PyInstaller로 단일 `.app` 번들 생성
- ffmpeg 의존성: 앱 번들 내 포함 (정적 바이너리) — 사용자가 별도 설치 불필요
  - 개발 환경: `brew install ffmpeg`로 시스템 ffmpeg 사용
- Whisper 모델: 최초 실행 시 자동 다운로드 (~1.5GB), `~/.cache/whisper/` 저장

### 4.4 신뢰성

- 5개 샘플 파일(각 10분~1시간) 연속 변환 시 크래시 없음
- 변환 중 앱 강제 종료 후 재시작 시 정상 동작 (이전 임시 파일 자동 정리)
- 에러 발생 시 앱 전체 중단 없이 해당 작업만 실패 처리

## 5. 출력 형식 명세

### 5.1 .txt 파일 예시

```
[00:00:00] 안녕하세요, 오늘 회의를 시작하겠습니다.
[00:00:05] 첫 번째 안건은 3분기 실적 검토입니다.
[00:00:12] Q3 revenue was up 15% compared to last year.
[00:00:18] 감사합니다. 다음 안건으로 넘어가겠습니다.
```

### 5.2 타임스탬프 형식

- 형식: `[HH:MM:SS]` (시간:분:초, 항상 2자리)
- Whisper 세그먼트 단위로 줄바꿈
- 빈 세그먼트(무음 구간, 공백만 포함) 제외
- 인코딩: UTF-8 (BOM 없음)
- 줄바꿈: LF (`\n`)

### 5.3 파일명 규칙

- 원본: `{name}.{ext}` → 출력: `{name}.txt`
- 예시: `meeting_20260405.m4a` → `meeting_20260405.txt`
- 특수문자 포함 파일명도 그대로 유지 (OS 허용 범위 내)
